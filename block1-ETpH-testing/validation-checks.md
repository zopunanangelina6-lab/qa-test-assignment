 Проверки валидации полей (по XSD схеме)

На основе XSD-схемы форматов электронных документов ФНС определены ключевые проверки валидации полей ЭТрН.

 1. ИНН (Идентификационный номер налогоплательщика)

- Длина: ровно 10 цифр для ЮЛ или 12 цифр для ФЛ
- Тип данных: только цифры (0-9)
- Контрольное число: проверка корректности по алгоритму ФНС
- Не начинается с 000 (невалидный префикс)
- Соответствие типу участника:
- Грузоотправитель-ЮЛ → ИНН 10 цифр
- Водитель-ФЛ → ИНН 12 цифр

Негативные сценарии:
- ИНН из 9 цифр → ошибка валидации
- ИНН из 11 цифр для ЮЛ → ошибка
- Буквы в ИНН → ошибка
- ИНН с некорректным контрольным числом → ошибка


2. КПП (Код причины постановки на учёт)

- Длина: ровно 9 символов
- Формат: XXXXXYYYY (5 цифр код налогового органа + 4 цифры причина)
- Тип данных: только цифры
- Присутствует только для ЮЛ (у ФЛ КПП отсутствует)

Негативные сценарии:
- КПП из 8 или 10 символов → ошибка
- Буквы в КПП → ошибка
- КПП у физического лица → ошибка валидации

 3. Вес груза

- Тип данных: числовое значение (decimal)
- Больше 0 (нулевой вес недопустим)
- Точность: до 3 знаков после запятой
- Единица измерения: килограммы (обязательно указана)
- Не превышает грузоподъёмность ТС

Негативные сценарии:
- Вес = 0 → ошибка
- Вес < 0 → ошибка
- Вес = "abc" → ошибка типа данных
- Вес превышает max ТС → предупреждение/ошибка


 4. Дата создания ЭТрН

- Формат: ДД.ММ.ГГГГ (ISO 8601)
- Не может быть будущей (≤ текущей даты)
- Не может быть раньше даты регистрации компании-отправителя
- Корректность календарной даты (не 31.02.2026)

Негативные сценарии:
- Дата завтрашнего дня → ошибка
- Несуществующая дата (30.02.2026) → ошибка
- Пустая дата → ошибка обязательного поля


5. Номер транспортной накладной

- Уникальность в системе (не дублируется)
- Формат: XX-XXXXXX (префикс + номер, согласно XSD)
- Не может быть пустым
- Длина: соответствует XSD-схеме

Негативные сценарии:
- Дублирующийся номер → ошибка уникальности
- Пустой номер → ошибка обязательного поля
- Неверный формат → ошибка валидации


6. Государственный регистрационный номер ТС

- Формат: А000АА 000 (1 буква + 3 цифры + 2 буквы + 2-3 цифры региона)
- Допустимые буквы (только те, что совпадают с латиницей): А, В, Е, К, М, Н, О, Р, С, Т, У, Х
- Код региона: существует в справочнике регионов РФ (01-92, 190-199)
- Не может быть пустым

Негативные сценарии:
- Буквы вне допустимого списка (например, "Ж", "Ш") → ошибка
- Несуществующий код региона → ошибка
- Пустой госномер → ошибка

 7. Данные водителя

- ФИО: три отдельных поля (Фамилия, Имя, Отчество)
- Номер водительского удостоверения: формат XXXX XXXXXX
- Дата выдачи прав: не будущая, не старше 10 лет (права действительны 10 лет)
- Категория прав: соответствует типу ТС (B, C, D, CE и т.д.)

Негативные сценарии:
- Просроченные права → ошибка
- Категория прав не соответствует ТС → ошибка
- Пустое ФИО → ошибка

---

### 8. Реквизиты сторон (Грузоотправитель, Грузополучатель, Перевозчик)

- ИНН/КПП — валидация по правилам выше
- Наименование ЮЛ: не пустое, длина по XSD
- Адрес: заполнен, формат соответствует ФИАС/КЛАДР
-  Контактные данные: телефон в формате +7 (XXX) XXX-XX-XX

Негативные сценарии:
- Пустое наименование → ошибка
- ИНН не проходит проверку ФНС → ошибка
- Некорректный адрес → предупреждение

---

### 9. Электронная подпись (КЭП)

- Действительность сертификата на момент подписания
- Сертификат не отозван (проверка по списку отзыва)
  - Соответствие подписанта участнику ЭТрН (ФИО/ИНН совпадают)
- Целостность документа после подписания (хэш совпадает)

Негативные сценарии:
- Просроченный сертификат → отказ в подписании
- Отозванный сертификат → отказ
- Подпись не соответствует участнику → отказ
- Изменённый после подписания документ → аннулирование подписи


 Общие проверки целостности

-  Все обязательные по XSD поля заполнены
-  Ссылочная целостность: все ID участников существуют в системе
-  Временная целостность: даты не противоречат друг другу (дата создания ≤ дата подписания ≤ дата перевозки)
-  Суммарный вес всех грузов = заявленному в шапке
