 Чек-лист тестирования ЭТрН

 1. Создание ЭТрН (Happy Path)
-  Создать новую ЭТрН с валидными данными
-  Заполнить обязательные поля:
-  Грузоотправитель (ИНН, КПП, наименование)
-  Грузополучатель (ИНН, КПП, наименование)
-  Перевозчик (ИНН, данные ТС)
-  Добавить информацию о грузе:
-  Наименование груза
-  Вес (кг)
-  Объем (м³)
-  Количество мест
-  Указать транспортное средство:
-  Марка ТС
-  Госномер
-  Данные водителя (ФИО, права)
-  Сохранить черновик
-  Отправить на подпись грузоотправителю

 2. Валидация обязательных полей
-  Проверка заполнения ИНН (10/12 цифр)
-  Проверка формата КПП (9 символов)
-  Проверка даты создания (не может быть будущей)
-  Проверка веса груза (> 0, числовое значение)
-  Проверка уникальности номера накладной
-  Проверка обязательности всех помеченных полей

 3. Смена статусов ЭТрН
-  Черновик → На подписании
-  На подписании → Подписана (после ЭП грузоотправителя)
-  Подписана → В пути (после начала перевозки)
-  В пути → Доставлена (после получения грузополучателем)
-  Отмена/аннулирование (если перевозка не состоялась)

 4. Интеграции
-  Подписание квалифицированной электронной подписью (КЭП)
-  Синхронизация с ФНС/ФТС
-  Обмен данными между участниками перевозки
-  Проверка статусов в реестре ЭТрН

5. Отображение реквизитов
-  Корректное отображение всех реквизитов сторон
-  Печать формы ЭТрН
-  Выгрузка в PDF/XML

 6. Негативные сценарии
-  Попытка создать ЭТрН с невалидным ИНН
-  Попытка подписать ЭТрН недействительной КЭП
-  Попытка изменить ЭТрН после подписания
-  Попытка аннулировать уже доставленную ЭТрН
-  Превышение максимального веса груза
