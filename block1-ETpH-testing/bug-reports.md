 Баг-репорты

Bug 1
Заголовок: Система принимает будущую дату в поле "Дата создания ЭТрН"

Severity: Medium  
Priority: High  
Окружение: Web-интерфейс создания ЭТрН, Chrome 120

Шаги воспроизведения:
1. Открыть форму создания новой ЭТрН
2. В поле "Дата создания" выбрать дату завтрашнего дня (или любую будущую)
3. Заполнить остальные обязательные поля валидными данными
4. Нажать "Сохранить"

Ожидаемый результат:  
Система не позволяет выбрать будущую дату. Отображается сообщение об ошибке валидации: "Дата создания не может быть позже текущей даты".

Фактический результат: 
ЭТрН успешно создаётся с будущей датой. Ошибки валидации не отображаются.

Влияние на бизнес:  
- Нарушение законодательных требований к оформлению транспортных документов
- Юридическая некорректность документа
- Возможные проблемы при проверке ФНС/ФТС

Воспроизводимость: 10/10 (100%)


Bug 2
Заголовок: При выходе участника из семейной подписки его плейлисты не синхронизируются с личным аккаунтом — данные теряются

Severity: Critical  
Priority: Blocker  
Окружение: Backend микросервис «Плейлисты» + «Подписки», API

Шаги воспроизведения:
1. Создать семейную подписку Яндекс Плюс с 3 участниками (A — владелец, B и C — участники)
2. Участник B создаёт персональный плейлист "Моя музыка" с 50 треками
3. Участник B выходит из семьи (через настройки подписки)
4. Авторизоваться под аккаунтом участника B
5. Перейти в раздел "Мои плейлисты"

Ожидаемый результат: 
Плейлист "Моя музыка" и все персональные данные (история прослушиваний, рекомендации) синхронизируются с личным аккаунтом участника B и остаются доступны.

Фактический результат:  
Плейлист "Моя музыка" остаётся привязанным к семейному аккаунту владельца A. Участник B теряет доступ к своим персональным данным после выхода из семьи. Данные не переносятся в личный аккаунт.

Влияние на бизнес: 
- Потеря пользовательских данных — критично для UX
- Массовые жалобы от пользователей семейных подписок (~25% платящей базы)
- Репутационные потери, негативные отзывы в сторах
- Риски оттока пользователей после выхода из семьи

Дополнительная информация:  
Баг выявлен при тестировании микросервиса «Подписки». Проблема в логике обработки Kafka-события при смене статуса подписки — данные не пересылаются в личный аккаунт участника.

Воспроизводимость: 10/10 (100%)
