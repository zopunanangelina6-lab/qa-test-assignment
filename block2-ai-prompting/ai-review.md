 ✅ Что ИИ сделал хорошо

2. Базовое покрытие: Покрыты основные happy path сценарии и базовая валидация полей (пустые значения, превышение лимитов).
3. Граничные значения: Корректно определены границы для веса (20000) и объема (82).

 ❌ Что ИИ упустил (Специфика логистики, UX и интеграций)

 1. Специфика логистики и бизнес-логики
- Габариты vs Вес: ИИ проверяет только вес и объем по отдельности. В логистике важен *объемный вес* (например, легкий, но огромный груз может не влезть в машину).
- Тип ТС: Не учтено, что для разных весов/объемов требуются разные типы машин (Газель vs Фура). API должен возвращать не просто цену, а и подходящий тип ТС.
- Запретные зоны: Нет проверки на доставку в закрытые/труднодоступные районы (например, районы Крайнего Севера зимой).

 2. UX и работа с сетью (Мобильный/Нестабильный интернет)
- Debounce автокомплита: ИИ не учел, что при быстром вводе названия города запросы не должны улетать на каждый символ. Нужна задержка (debounce ~300мс).
- Обработка таймаутов: Что если API расчета стоимости думает больше 10 секунд? Нужен спиннер и понятное сообщение, а не вечная загрузка.
- Потеря фокуса: При выборе города из списка фокус должен корректно переходить к следующему полю (важно для UX).

 3. Интеграции и API (Backend)
- Идемпотентность: Если пользователь дважды быстро кликнет "Рассчитать стоимость", фронтенд должен блокировать кнопку, чтобы не отправить два одинаковых POST-запроса.
- Кэширование: Популярные маршруты (Москва -> Питер) должны кэшироваться, чтобы не дергать API каждый раз.
- Валидация на бэкенде: ИИ упомянул это, но не дал конкретного кейса: что если обойти фронтенд и отправить через Postman вес 25000 кг? Бэкенд обязан вернуть `422 Unprocessable Entity`.


 Избыточные тесты (Что я бы удалила)

1. "Цвет кнопки #007BFF" и "Шрифт 14px" — это косметические проверки UI. Для MVP и функционального тестирования это "мусор", который только замедляет регресс.
2. "Работа формы при отключенном JavaScript" — если фронтенд написан на React/Vue/Angular, то без JS форма просто не отрендерится. Тест бессмысленен.
3. "Копирование/вставка в поля" — стандартное поведение браузера, тестировать это отдельно не имеет смысла, если только нет специфичных масок ввода.
